* ESPHome 在线编译平台

基于 Vue 3 的单页前端将 ESPHome YAML 直接发送至 GitHub Actions Workflow 进行固件编译，无需自建后端。

* 目录结构
  - `frontend/` : Vite + Vue 3 前端应用
  - `.github/workflows/esphome-compile.yml` : 负责执行编译的 GitHub Actions Workflow

* 前端开发
  #+begin_src shell
  cd frontend
  npm install   # 或者 pnpm install / yarn install
  npm run dev
  #+end_src

  浏览器访问 `http://localhost:5173`。界面需要填写：
  - GitHub Personal Access Token（必须包含 `repo` 与 `workflow` 权限，令牌仅保存在浏览器端）
  - 仓库拥有者、仓库名、Workflow 文件名（默认 `esphome-compile.yml`）与触发分支
  - ESPHome YAML
  - 可选：设备名称、板子类型（仅用于日志展示）

  点击 *提交编译* 后，前端会：
  1. 将 YAML 做 Base64 编码并调用 `workflow_dispatch` 接口；
  2. 轮询 Workflow 运行状态；
  3. 在运行完成后展示可下载的固件压缩包。

* GitHub Actions Workflow

  `esphome-compile.yml` 会：
  1. 接收 Base64 编码的 YAML，并写入 `config.yaml`；
  2. 安装最新的 ESPHome；
  3. 执行 `esphome compile config.yaml`；
  4. 上传 `.esphome/build` 目录与原始配置为 Artifact。

  Workflow 使用 `run-name` 携带请求 ID，前端依赖该字段关联状态，请不要移除。

* 注意事项
  - GitHub Personal Access Token 建议设置短期有效期并仅限必要仓库；
  - 浏览器端存储了仓库信息（不含 Token），可以随时清除；
  - 如果 Workflow 失败，可在 GitHub Actions 页面查看日志定位问题。
